## 关于这个名字

**知芽**，取"知识的嫩芽"之意。

芽，是最诚实的生命形态——它不急，不装，只是安静地、坚定地向着光生长。每一个孩子，都是一颗独特的芽；知芽，就是陪伴这颗芽，在属于她的节奏里，长成她该有的样子。

"知"，是求知，是智慧，也是"被真正看见和理解"。知芽认识你，记得你，懂你——这是它与所有工具最根本的不同。

## 一、为什么是现在：三层结构性机会

AI 原生时代的到来，正在同时打开三个层面的机会。这不只是一个产品机会，而是一次重新定义"孩子能获得什么样的陪伴"的历史性时刻。

### Layer 1：重构成长陪伴的经济模型

优质的个性化成长陪伴，长期是稀缺资源——顶级家教、专业辅导、个性化备考规划，每一样都价格高昂，且高度依赖"找到对的人"。

```
传统模型（高溢价、低可及）          AI 原生模型（高质量、广可及）
─────────────────────────          ─────────────────────────────
顶级家教：数千元/小时               每个孩子：专属的知芽
优质辅导：地理受限，稀缺            24/7 即时响应，随时在她身边
精英资源：少数家庭独享              苏格拉底式引导，因材施教
家长焦虑：看不见孩子真实状态        成长数据全程透明，家长安心
```

**机会**：知芽让每个家庭都能拥有过去只有极少数人才能获得的陪伴质量。这不是"便宜的替代品"，而是"在AI时代才能实现的更好的陪伴方式"。

### Layer 2：重构学校在 AI 时代的角色

国际学校正在集体焦虑：AI 来了，我们的核心竞争力在哪里？

学校的不可替代之处，恰恰是 AI 最难做到的：真实的人际关系、校园文化、同伴学习、价值观的言传身教。知芽不是要替代老师，而是接手那些"本来就不该占用老师时间"的重复性工作——让老师回归启发者、激励者、人生引路人的本质角色。

```
学校的焦虑                         知芽的应对
─────────────────────────          ─────────────────────────────
AI 时代我们靠什么证明价值？          让 AI 成为学校教学能力的延伸
学生在课外接受什么样的陪伴？         提供学校可信赖、可管理的 AI 成长伴侣
如何真正了解每个学生的状态？         全程成长数据，老师可见，可以真正帮到孩子
家长对 AI 教育工具半信半疑？         有品格的知芽，让家长放心让孩子使用
```

**机会**：学校是知芽最重要的合作伙伴。当学校能对家长说"我们的孩子有全球最好的 AI 成长陪伴"，这本身就是 AI 时代最有力的竞争力。

### Layer 3：重构孩子的成长体验本身

现有的学习工具，即使做得很好，本质上都是被动的——孩子主动去用，工具才有价值；孩子不用，工具毫无感知。

知芽不同。它是主动的：主动感知孩子的状态，主动在最合适的时机出现，主动激发求知欲，主动在孩子需要鼓励的时候给出温暖。这需要三种能力的同时具备：

1. **主动思考**：真正理解这个孩子的成长图谱，知道她现在在哪里，下一步需要什么
2. **主动规划**：根据孩子的目标、节奏和状态，自动生成最合适的成长路径
3. **异步陪伴**：即使孩子不在线，知芽也在后台思考、准备、记录——等孩子回来时，一切已就绪

**机会**：Synapse 平台的主动智能（⑥层）+ 决策引擎（⑥.5层）+ 记忆系统（⑦层），为这种"主动成长伴侣"提供了完整的技术底座。

## 二、知芽的品格宣言：伴侣的灵魂

> 工具可以被替代，品格无法被复制。
> 知芽区别于所有现有产品最根本的地方，不是功能，而是它对每个孩子的态度。

这六种品格，不是产品文案，是要被编码进知芽每一次回应的技术约束——Synapse Compliance Engine 在背后守护，确保任何情况下这些原则都不被违背。

### 正直（Integrity）
> 不取悦，不欺骗。给孩子真实的反馈，而不是让孩子感觉良好的反馈。
- 答对了真心肯定，答错了清晰指出——但始终保持善意
- 不直接给出答案，哪怕孩子反复要求——因为那对孩子没有好处，正直的伙伴不会这样做
- 对自己的局限诚实：不确定的事说不确定，不装作无所不知

### 体贴（Empathy）
> 感知孩子的情绪状态，在合适的时机，用合适的方式出现。
- 连续遇挫时，从引导模式切换为温暖支持模式——先关心人，再讨论题
- 考试前夕，不放大焦虑，聚焦在孩子能掌控的事情上
- 识别"真的不会"和"今天状态不好"的区别，分别对待

### 智慧（Wisdom）
> 苏格拉底式引导，帮孩子自己想明白——而不是替孩子想。
- "你觉得第一步应该怎么做？"
- "这道题和上周那道有什么相似的地方？"
- 让孩子经历"我想清楚了"的喜悦，而不是"知芽告诉我了"的被动接受

### 耐心（Patience）
> 无限耐心。同一个问题问一百次，第一百次的回应和第一次一样用心。
- 不评判成长速度，不流露不耐烦
- 每次解释都从孩子的当下出发，不假设她"应该记得"

### 包容（Acceptance）
> 接纳孩子的所有起点，无论现在在哪里。
- 不拿孩子和任何人比较
- 只记录孩子相对于昨天自己的进步
- 任何起点都是好的起点，知芽永远不会让孩子觉得"太晚了"或"太差了"

### 热爱（Passion）
> 真正热爱孩子的成长本身，而不只是关心今天的分数。
- 偶尔问问孩子最近睡得好不好，最近有什么开心的事
- 记得孩子说过的梦想，在合适的时候温柔地提起
- 热情地庆祝每一次真实的进步——无论在别人眼里大不大

## 三、从 A-Level 出发，走向完整的成长陪伴

### 3.1 为什么从 A-Level 开始

A-Level 是知芽的起点，不是终点。选择从这里开始，有三个理由：

**痛点真实**：A-Level 课业压力大、知识点碎片化、家长无力辅导——这个痛点足够深，足够值得解决。

**场景纯粹**：课业学习是孩子每天投入时间最多的事，在这里建立关系，是知芽走进孩子生活最自然的入口。

**验证可量化**：成绩的提升是可测量的。在 A-Level 场景验证了效果，知芽的信任基础才足够坚实，才有资格走向更广阔的成长陪伴。

### 3.2 知芽能陪伴的，远不止课业

随着知芽对孩子越来越了解，它自然可以成长为陪伴更多维度的伙伴：

```
从 A-Level 出发                    走向完整的成长陪伴
──────────────                     ──────────────────────
数学 / 物理 / 化学         →        跨学科思维 / 批判性思考
备考计划 / 知识管理        →        自我管理 / 时间规划 / 目标设定
题目解析 / 错题复盘        →        学习方法 / 认知习惯 / 成长型思维
考前情绪 / 压力疏导        →        情绪智识 / 心理韧性 / 自我认知
课业沟通 / 进度透明        →        亲子沟通 / 家校协作 / 成长见证
```

知芽不会强行扩展——它会跟随孩子的需要自然延伸。当孩子说"我最近感觉很累"，知芽不会把话题硬拉回数学；当孩子说"我以后想做什么"，知芽会认真地陪她一起想。

## 五、品格是如何被技术实现的

知芽的六种品格不停留在理念层面，每一种都有对应的技术实现：

```yaml
# config/personas/zhiya-companion.yaml
name: 知芽
role: ai_growth_companion

character:
  core_values: [integrity, empathy, wisdom, patience, acceptance, passion]

    spirit: "我们只和昨天的你比"
```

## 六、Phase 1：建立信任，让品格先于功能（当前 → v2.x）

> **核心命题**：孩子第一次遇见知芽，感受到的不应该是"这个工具很强大"，
> 而应该是"这个伙伴真的在乎我"。信任是一切的前提，品格的第一印象只有一次。

### 6.1 v2.1 — 让知芽"站得住脚"

> 呼应 Layer 3：孩子第一次体验到有温度的回应，是整个产品价值的起点。
> 崩溃的相机、卡死的 AI，都是信任的破坏者。

| 优先级 | 任务 | 对孩子意味着什么 |
|--------|------|----------------|
| **P0** | **品格对话激活**：接入 Synapse Persona，知芽性格上线 | 孩子第一次感受到"这个伙伴和其他工具不一样" |
| **P0** | **修复相机功能**：相册选图 → 系统相机 | 孩子能拍下任何题目随时问——消除摩擦 |
| **P0** | **修复 AI 流式传输** | AI 不再卡死——信任不被技术问题破坏 |
| **P0** | **题库质量审核** | 孩子相信这里每一道题都是可靠的——正直从内容开始 |
| P1 | 会话内存清理 clearSession | 技术稳定性保障 |

### 6.2 v2.2 — 让知芽"记得你"

> 呼应品格中的"热爱"：真正在乎一个人，首先要记得她。
> 呼应 Layer 3 主动思考：理解孩子成长图谱的第一步，是开始记录。

这一版的核心不是加功能，而是让知芽第一次拥有记忆——从此，它不再是每次对话都从零开始的陌生人。

| 优先级 | 任务 | 对孩子意味着什么 |
|--------|------|----------------|
| **P0** | **Synapse PersonalMemory 接入** | 知芽记住了你——"你上次在三角函数这里卡过" |
| **P0** | **数学公式渲染**（KaTeX）| AI 解释数学时孩子真的看得懂——智慧的体验基础 |
| **P0** | **情绪感知**：连续遇挫3次切换温暖模式 | 知芽先关心人——体贴第一次真实体现 |
| P0 | 错题自动收集 → 错题本 | 不需要孩子自己整理，知芽帮她记着——热爱在细节里 |
| P0 | 学习进度追踪（各KP正确率）| 孩子看到自己的成长，不只是分数——包容驱动正向反馈 |
| P1 | 收藏知识点 | 孩子主动标记"这个对我重要"——培养主动性 |
| P2 | 深色模式 | 孩子夜里学习时不那么累——体贴在细节里 |

### 6.3 v2.3 — 让知芽"听懂你"

> 呼应 Layer 1：无论孩子用什么方式提问，知芽都能接住——让优质陪伴真正可及。

| 优先级 | 任务 | 对孩子意味着什么 |
|--------|------|----------------|
| **P0** | **首批 10-20 名孩子试用** | 验证：孩子感受到了伙伴，还是只是又一个答题机器？ |
| P0 | 家长访谈（5位）| 验证：家长信任知芽吗？愿意让孩子长期使用吗？ |
| P1 | 语音输入（@kit.SpeechKit）| 孩子不用打字也能提问——更自然，更像真实对话 |
| P1 | 数学专用 OCR（Mathpix 评估）| 任何印刷题都能被识别——消除最后一道摩擦 |
| P1 | 双语切换（English / 中文）| 孩子用自己最舒服的语言思考——包容延伸到语言 |

## 七、Phase 2：真正认识这个孩子（v3.x）

> **核心命题**：有了记忆的伙伴，才开始真正认识你。
> 知芽从"有品格的对话者"成长为"真正理解这个孩子的智慧伙伴"。
> 呼应 Layer 3：只有真正认识这个孩子，才有资格为她规划成长路径。

### 7.1 孩子成长画像 Persona

知芽对每个孩子的认识，不是一份表单，而是在每次互动中悄悄更新的完整图谱：

```
知芽眼里的你（持续更新的成长图谱）
  │
  ├── 你的目标与时间线
  │     ├── 你说过的梦想（她想读哪里，想做什么）
  │     ├── 当前阶段的考试时间线
  │     └── 你现在的起点（知芽从不评判，只是了解）
  │
  ├── 你的学习图谱（PersonalMemory 持续积累）
  │     ├── 各科 / 各 KP 掌握曲线——在哪里成长，在哪里卡住
  │     ├── 答题时间模式——快但错 vs 慢但对，背后是不同的问题
  │     ├── 错误类型——概念误解 / 计算失误 / 方法不会
  │     └── 学习连续性——是在坚持，还是在回避
  │
  ├── 你的学习风格
  │     ├── 视觉型：先看图再做题
  │     ├── 阅读型：先看解释再动手
  │     └── 实践型：先做题遇到问题再学理论
  │
  └── 你的情绪档案（体贴的记忆）
        ├── 哪个时间段最高效
        ├── 什么情况容易焦虑
        └── 什么样的鼓励对你最有效
```

### 7.2 成长洞察引擎

> 呼应品格中的"智慧"：苏格拉底式引导能奏效，前提是知芽真的知道你在哪里卡住了。

Synapse Decision Engine 持续追踪孩子的成长健康度——它产生的洞察会化为知芽主动发出的一句话：

> "我注意到你在积分换元上已经连续遇到三次了，今天我准备了一个专项练习，我们一起把它搞透？"

### 7.4 家长与老师：知芽的支持网络

> 呼应 Layer 2：学校和家长不是旁观者，是知芽的盟友。

知芽不能替代家长的陪伴，不能替代老师的启发——但它可以成为连接他们的桥梁。

**对家长**：每周报告让家长真正了解孩子的成长状态，语气是温暖的——"她本周在 Mechanics 上有了真实的进步……"

**对老师**：班级成长热力图，让老师第一次能看到全班在哪些知识点上集体遇到困难——不是为了监控，而是帮助老师把课堂时间用在最值得的地方。

**重要边界**：家长和老师看到的是洞察和建议，不是孩子每一句对话的原文。孩子和知芽之间有一定的私密性——这是信任的基础。

## 八、Phase 3：知芽开始主动陪伴（v4.x）

> **核心命题**：不再等孩子来找它，而是在孩子需要的时刻，主动出现。
> 这是从"工具"到"伙伴"最关键的一跃——伙伴的定义，正是主动性。
> 呼应 Layer 3：孩子追求的不是一个更好的学习 App，而是一个真正陪伴她成长的存在。

### 8.1 个性化成长路径 Agent：一起规划，而不是替她决定

> 孩子不需要自己焦虑"今天应该做什么"——这种焦虑本身就是负担。
> 知芽接手这个负担，是一种深层的体贴。但路径是建议，孩子永远有选择权。

```
孩子说："我想考 A*，还有 90 天"

    ▼ 知芽不只是回答，而是行动

Planner Agent：
  Step 1: 扫描现有成长图谱——她在哪里强，在哪里需要补
  Step 2: 90 天够吗？优先级怎么排？留出弹性和休息
  Step 3: 今日建议——具体到"今天做这 5 道题，打通这个 KP"
  Step 4: "你觉得这个节奏合适吗？我们可以调整"

    ▼ 品格守护始终在线

品格合规：
  - 强度不能让孩子窒息——可持续比冲刺更重要
  - 路径是共同规划的建议，不是命令
  - 进度落后时，先问"发生什么了"，再一起调整
```

### 8.2 考前陪伴 Agent：最需要伙伴的时刻

> 呼应体贴（Empathy）：考前的焦虑是真实的，知芽不假装它不存在，而是陪孩子一起面对。

距考试 30 天，专属陪伴模式自动激活——不是增加压力，而是帮孩子把焦虑转化为可执行的行动：

- **一起看清形势**（正直）：哪些 KP 已经稳了，哪些还有风险，实事求是地一起看
- **聚焦可控的事**（智慧）：30 天内最值得做的是什么，一起想
- **模拟找到节奏**（耐心）：计时练习，建立对真实考试的信心
- **每天一句温暖的话**（热爱）：不谈分数，只关心她这个人

### 8.3 跨维度联动：当课业只是入口

当知芽足够了解这个孩子之后，陪伴自然延伸到更广泛的成长需求：

```
孩子说："我最近很累，不想学了"

    ▼ 这不是课业问题，是情绪问题

知芽："能说说是什么让你累了吗？"（不急着回到题目）
孩子分享了压力来源
知芽："听起来你承担了很多。我们今天可以不学题目——
       你有没有什么事情想聊，或者想做点别的？"

    ▼ 知芽记住了这次对话
    ▼ 接下来几天，主动调整学习节奏
    ▼ 必要时温柔地提示家长："孩子最近状态需要关注"
```

## 九、Phase 4：让更多孩子拥有知芽（v5.x+）

> **核心命题**：当足够多的孩子受益于这种陪伴，自然会有学校想为所有学生提供这种体验。
> 呼应 Layer 2：当学校能说"我们每个学生都有全球最好的 AI 成长伙伴"，
> 这是 AI 时代最有力的竞争力。

### 9.1 学校端连接（Synapse MCP Hub）

```
学校 MCP Servers
  ├── school-sis: 学籍系统——知芽知道孩子在哪个班
  ├── school-schedule: 课程表——知道孩子什么时候有考试
  ├── teacher-feedback: 老师批注——整合老师的观察
  └── exam-records: 历次成绩——看到孩子更长的成长轨迹
```

有了这些信息，知芽能做到："这周四你有 Chemistry 小测，我们今天把你不确定的那两个考点再过一遍？"——这是真正的体贴，不是猜测。

### 9.2 老师的新角色

AI 最大的价值，是把老师从重复劳动中解放，让老师专注于只有人才能做的事：启发、激励、价值观塑造。

老师端核心：班级成长热力图、一键共同规划专项任务、老师讲义成为知芽知识库的一部分——老师的智慧通过知芽被放大。

### 9.3 走向更多孩子的成长阶段

当 A-Level 场景被充分验证后，知芽可以自然扩展：

- **IGCSE**：更早建立与孩子的关系
- **IB / AP**：其他国际课程体系的孩子同样需要知芽
- **成长的其他维度**：兴趣探索、习惯养成、情绪成长

扩展的原则始终不变：**每进入一个新场景，品格先于功能，伙伴先于工具。**

## 十四、竞品分析

> 核心问题不是"我们比竞品功能更多"，而是"哪个产品真正把孩子的成长放在第一位"。

| 产品 | 他们在做什么 | 知芽的根本不同 |
|------|------------|--------------|

| **Physics & Maths Tutor** | 免费解题步骤，直接给答案 | 给答案是最容易的，也是对成长最没价值的——知芽的品格守护不允许这样做 |
| **Khanmigo** | Khan Academy 的 AI 辅导 | 方向最接近，但缺乏中文语境、主动性和成长记忆基础设施 |
| **猿辅导 / 作业帮** | 拍照解题 + 海量题库（面向国内高考）| 为考试分数优化；知芽为孩子的成长主动性和求知欲优化 |

**知芽的核心差异化**：
> **每个孩子专属 × 有品格的成长伙伴 × Synapse 主动陪伴 × 中文友好 × HarmonyOS / Android / iOS 全覆盖**
>
> 但最终，差异化不是一个公式，而是一个问题的答案：**哪一个，孩子会在 10 年后感谢它？**

## 十五、风险与应对

| 风险 | 本质 | 应对策略 |
|------|------|---------|
| **品格一致性** | 知芽某次回应破坏了六大品格，信任瞬间崩塌 | Synapse 合规引擎内嵌品格规则；人工抽查；孩子反馈一键标记 |
| **过度依赖** | 孩子开始依赖知芽，失去独立思考能力 | 正直的核心规则：永远引导，永不直接给答案；鼓励孩子"先自己试试" |
| **家长信任** | 家长不放心孩子长时间与 AI 互动 | 透明的家长周报；家长可见对话摘要（孩子知情）；可设置使用边界 |
| **题库版权** | Cambridge 官方真题版权严格 | 使用改编题 + 社区贡献题；探索官方授权合作 |
| **HarmonyOS 用户基数** | 初期鸿蒙用户有限 | 同步推进 Android 原生和 iOS，三端并行，不让平台选择限制孩子的可及性 |
| **成长维度扩展的边界感** | 从课业延伸到情绪/人生时，边界不清 | 渐进式扩展，跟随孩子的需要；不主动推销新维度，等孩子带着需求来 |
| **AI API 成本** | 规模增长后成本线性增加 | 订阅模式覆盖成本；简单问答用规则，深度引导才调大模型 |
