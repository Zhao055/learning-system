# A-Level Helper — 产品愿景与路线图

## 一句话定位

> **A-Level Helper 是面向国际学校 A-Level 学生的 AI-native 学习伴侣，从结构化刷题出发，最终成为每个孩子的个性化智能学伴。**

---

## 背景与洞察

### 用户痛点

| 角色 | 痛点 |
|------|------|
| **学生** | A-Level 科目多、知识点碎片化；课后缺乏即时辅导；刷题缺乏针对性，反复做已会的题 |
| **家长** | 不了解孩子学习进度和薄弱点；国际课程体系陌生，无法辅导 |
| **老师** | 难以追踪每个学生的知识掌握情况；出题和批改耗时 |

### 市场机会

- 中国国际学校 A-Level 学生群体持续增长，但专门的 A-Level 学习工具稀缺
- 现有工具（Save My Exams、Physics & Maths Tutor）都是 Web 端，无移动原生体验
- HarmonyOS NEXT 生态早期，教育赛道几乎空白，先发优势明显
- Synapse AI 平台提供了现成的 Agent 基础设施，可以低成本实现 AI-native 体验

## Phase 2: 智能化学习（v3.x）

**目标：** 引入数据驱动能力，让 AI 不只是"问答机器人"，而是能理解学生学习状态的智能体。

### v3.0 — 学习数据引擎
- [ ] 用户登录 & 数据持久化（本地数据库 → 云同步）
- [ ] 学习数据模型设计：
  ```
  StudentProfile
    ├── 每科/每Paper/每KP 的正确率
    ├── 答题时间分布
    ├── 错题模式分析（哪类错误最频繁）
    └── 学习频率和连续性
  ```
- [ ] 学习报告页面（日/周/月维度）
- [ ] 薄弱知识点自动识别

### v3.1 — AI 增强
- [ ] **智能出题**：基于薄弱知识点动态生成练习题（AI 生成 + 质量校验）
- [ ] **讲题模式**：AI 不只给解释，而是苏格拉底式引导（"你觉得下一步应该做什么？"）
- [ ] **知识图谱可视化**：展示 KP 之间的关联，点击探索
- [ ] **拍照解题增强**：识别题目后匹配已有题库，补充变式练习

### v3.2 — 家长/老师视角（轻量版）
- [ ] 班级码加入机制
- [ ] 老师端：查看学生学习报告、布置练习
- [ ] 家长端：查看孩子学习概况（周报推送）

## Phase 3: AI-Native 学伴 — Synapse 赋能（v4.x）

**目标：** 接入 Synapse Agent 平台，从"工具"变为"学伴"——AI 主动、个性化、持续陪伴。

### 核心理念

借鉴 Synapse 的五大原则在教育场景的映射：

| Synapse 原则 | 教育场景映射 |
|-------------|-------------|
| **解放人** | 把学生从低效重复刷题中解放，专注理解和思考 |
| **消除流程摩擦** | 不需要学生自己规划学什么，AI 自动安排最优路径 |
| **数据驱动决策** | 基于学习数据精准定位问题，不凭感觉刷题 |
| **聚焦价值创造** | 从"做100道题"转向"真正掌握10个知识点" |
| **跨时代竞争力** | 当其他学生还在传统刷题，这些学生已有 AI 学伴 |

### v4.0 — Synapse Agent 接入

**架构集成：**
```
A-Level Helper App (HarmonyOS / 多端)
       │
       ▼
  Synapse Gateway (Hono API)
       │
       ├── Student Persona (学生角色定义)
       │     ├── 学科偏好
       │     ├── 学习风格（视觉/听觉/读写）
       │     ├── 目标成绩
       │     └── 考试时间线
       │
       ├── Agent Core (智能核心)
       │     ├── Planner: 制定个性化学习计划
       │     ├── Executor: 执行教学动作
       │     └── Tool Loop: 调用题库/知识库/生成工具
       │
       ├── Memory System (记忆系统)
       │     ├── PersonalMemory: 每个学生的学习历史
       │     ├── KnowledgeBase: A-Level 知识库
       │     └── OrgMemory: 班级/学校级别的学习模式
       │
       ├── Proactive Intelligence (主动智能)
       │     ├── 每日学习提醒和推荐
       │     ├── 考前冲刺计划自动生成
       │     ├── 薄弱知识点预警
       │     └── 学习模式异常检测（突然不学了）
       │
       └── Decision Engine (决策引擎)
             ├── 学习效率指标追踪
             ├── 知识掌握度评估
             └── 学习路径优化建议
```

### v4.1 — 个性化学习路径
- [ ] AI 根据目标成绩 + 当前水平 + 考试日期，自动规划每日学习任务
- [ ] 动态调整：做错了→加强同类练习；掌握了→跳过已会的
- [ ] "今日任务"首页：打开 App 就知道今天学什么
- [ ] 学习风格适配：有的学生适合先看知识点再做题，有的适合先做题再学

### v4.2 — 对话式学伴
- [ ] 不再是"问一答一"，而是 Agent 有持续记忆的长期对话
- [ ] AI 主动发起互动："你昨天的三角函数练习正确率只有40%，要不要我帮你复习一下？"
- [ ] 考前模式：Agent 自动分析薄弱点，生成专项冲刺计划
- [ ] 情绪感知：检测到学生连续答错时，切换为鼓励和引导模式

### v4.3 — 学习社区
- [ ] 学生之间的学习数据对比（匿名化）
- [ ] "今日挑战"：每天一道难题，全校排名
- [ ] 学习成就系统：连续打卡、满分记录、进步最快奖
- [ ] 学习小组：几个同学组队，互相激励

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 题库内容质量 | 错误的答案会失去用户信任 | 建立教师审核机制；标注"社区贡献" vs "官方审核" |
| HarmonyOS 用户基数小 | 初期用户获取困难 | 优先跨端方案；微信小程序扩展覆盖面 |
| AI API 成本 | 大量使用时费用增长 | 分级策略：简单问题用本地规则，复杂问题才调 AI |
| Synapse 集成复杂度 | 教育场景 Persona 需要深度定制 | Phase 3 之前积累足够的学习数据和用户理解 |
